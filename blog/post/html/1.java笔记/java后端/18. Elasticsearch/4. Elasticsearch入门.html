                                                                                                                                                                      ElasticSearch技术

                                                                                                                                                                      全文搜索技术是最常见的需求，目前Elasticsearch是全文搜索引擎的首选。维基百科、Stack Overflow、Github都使用它。

                                                                                                                                                                      • ElasticSearch底层使用开源库Lucene，它是Lucene的封装。

                                                                                                                                                                      基本概念

                                                                                                                                                                      1. Node与Cluster

                                                                                                                                                                      本质上Elastic是一个分布式数据库，允许多台服务器协同工作。

                                                                                                                                                                      • Node：单个Elastic实例称为一个节点。名字默认使用UUID，可以自行配置。
                                                                                                                                                                      • cluster：一组节点构成一个集群。

                                                                                                                                                                      2. Index（索引）

                                                                                                                                                                      Elastic会索引所有字段，通过反向索引查找数据。

                                                                                                                                                                      • index：
                                                                                                                                                                        1. （按阮一峰的博客来说）Elastic数据结构最顶层的单位叫做索引。
                                                                                                                                                                        2. 如果对比mysql，Elastic的一个Index就相当于一个数据库。
                                                                                                                                                                        3. 每个index名字必须小写。
                                                                                                                                                                      curl -X GET 'http://localhost:9200/_cat/indices?v'

                                                                                                                                                                      3. Document（文档）

                                                                                                                                                                      • document：
                                                                                                                                                                        1. Index里面的一条记录称为document。
                                                                                                                                                                        2. document使用json格式表示。
                                                                                                                                                                        3. document使用相同的结构有利于提高搜索效率。

                                                                                                                                                                      3.1 Mapping（映射）

                                                                                                                                                                      • Mapping：
                                                                                                                                                                        • mapping相当于mysql中的表
                                                                                                                                                                        • mapping中定义了每个字段、字段所使用的分词器等等。

                                                                                                                                                                      4. Shard（分片）

                                                                                                                                                                      5. Replica（复制分片）

                                                                                                                                                                      6. Field type（字段类型）

                                                                                                                                                                      • text： 字符串
                                                                                                                                                                      • keyword：不能分析，只能被精确匹配的字符串
                                                                                                                                                                      • date： 日期类型
                                                                                                                                                                      • long： 长整型
                                                                                                                                                                      • integer： 整型
                                                                                                                                                                      • short“ 短整型
                                                                                                                                                                      • double： 双精浮点
                                                                                                                                                                      • array： 数组
                                                                                                                                                                      • object： 对象类型，json嵌套
                                                                                                                                                                      • ip： ip类型
                                                                                                                                                                      • geo_point： 地理类型

                                                                                                                                                                      7. 分词器

                                                                                                                                                                      分词器是Elastic搜索的核心，功能如下：

                                                                                                                                                                      1. 对文档内容分词，创建倒排索引。
                                                                                                                                                                      2. 对搜索内容分词，再倒排索引中索引文档。

                                                                                                                                                                      8. 内置分词器

                                                                                                                                                                      • 标准分词器（standard analyzer）默认。
                                                                                                                                                                      • 英文分词器（english analyzer）。
                                                                                                                                                                      • 简单分词器：按空格分词、大写转小写处理。
                                                                                                                                                                      • 空格分词器：按空格分词、不做大小写处理。

                                                                                                                                                                      9. 外部分词器

                                                                                                                                                                      • 中文分词器（ik_maxword）：尽可能拆除更多的词。（南京市长江大桥：南京市/南京/市长/长江…）
                                                                                                                                                                      • 中文分词器（ik_smart)：已被分出的词不会再被其他词占有。（南京市长江大桥：南京市/长江大桥）

                                                                                                                                                                      10. 评分规则

                                                                                                                                                                      • TF（token frequency）：词频，频率高相关性越强。
                                                                                                                                                                      • IDF（inverse document frequency）：逆向文档频率，包含分词的document的数量。数量越多则相关性越弱。
                                                                                                                                                                      • TFNORM（token frequency normalize）：词频规格化，根据field长度做归一化，文档内出现的频率越高，field越短相关性越强。

                                                                                                                                                                      11. 倒排索引与正排索引

                                                                                                                                                                      12. 乐观锁和悲观锁

                                                                                                                                                                      每个文档都有一个version版本，

                                                                                                                                                                      搜索结果各个字段，hits、took、timeout

                                                                                                                                                                      • took：整个执行过程耗费时间（ms）。
                                                                                                                                                                      • hits：命中，返回查询结果和查询信息。
                                                                                                                                                                        • total：文档总数。
                                                                                                                                                                        • 默认返回前十个文档
                                                                                                                                                                        • _score_：相关性
                                                                                                                                                                        • _max_score：查询结果中score的最大值
                                                                                                                                                                      • shard：查询中参与分片的总数，以及这些分片成功了多少个失败了多少个。
                                                                                                                                                                      • timeout：是否超时。默认不会超时，而会阻塞。可以设置参数GET /_search?timeout=10ms。

                                                                                                                                                                      1.PUT：创建索引

                                                                                                                                                                      • 非结构化方式创建索引
                                                                                                                                                                      PUT /employee(索引库)/type(过时)/1(id)
{
    "setting": {
        "number_of_shards": 1,
        "number_of_replicas": 1
    }
}
                                                                                                                                                                      • 结构化方式创建索引

                                                                                                                                                                      name 字段分别指定了索引和查询的两个不同的中文分词器。索引时尽可能多的分词，查询时尽可能少分词。

                                                                                                                                                                      tag字段指定了空格分词器，对于tag还需要排序、聚合操作，所以把fielddata开启，允许正排索引。

                                                                                                                                                                      category_name字段仅仅作为一个关键字，进行分词处理反而产生歧义。

                                                                                                                                                                      PUT /shop
{
    "setting": {
        "number_of_shards": 1,
        "number_of_replicas": 1
    },
    "mapping": {
        "properties": {
            "id": {"type": "integer"},
            "name": {"type": "text", "analyzer": "ik_max_word", "search_analyzer": "ik_smart"},
            "tags": {"type": "text", "analyzer": "whitespace", "fielddata": true},
            "category_name":{"type": "keyword"}
        }
    }
}

                                                                                                                                                                      说明：以下指定了个别字段使用英文分词器，日期类型指定了format，可以支持解析多种类型的日期字符串格式。

                                                                                                                                                                      PUT /movie
{
  "settings": {
    "number_of_shards": 1,
    "number_of_replicas": 1
  },
  "mappings": {
    "properties": {
      "title":{"type": "text","analyzer": "english"},
      "tagline":{"type": "text","analyzer": "english"},
      "release_date":{"type": "date","format": "8yyyy/MM/dd||yyyy/M/dd||yyyy/MM/d||yyyy/M/d"},
      "popularity":{"type": "double"},
      "overview":{"type": "text","analyzer": "english"},
      "cast":{
        "type": "object",
        "properties": {
          "character":{"type":"text","analyzer":"standard"},
          "name":{"type":"text","analyzer":"standard"}
        }
      }
    }
  }
}
