                                                                                                                                                                      Maven

                                                                                                                                                                      1.Maven是什么

                                                                                                                                                                      是 Apache 下的一个 Java 开发的开源项目，基于项目对象模型（Project Object Model，缩写：POM）。 Maven利用一个中央信息片断能管理一个项目的构建、报告和文档等步骤。

                                                                                                                                                                      Maven 是一个项目管理工具，可以对项目进行构建、依赖管理。

                                                                                                                                                                      ​ 官网：http://maven.apache.org/ ​ maven坐标搜索网站：https://mvnrepository.com/

                                                                                                                                                                      2.Maven的作用

                                                                                                                                                                      • 项目依赖管理
                                                                                                                                                                      • 提供统一的模板，统一项目结构
                                                                                                                                                                      • 对项目创建、编译、打包、发布等操作进行管理

                                                                                                                                                                      4.Maven仓库和坐标

                                                                                                                                                                      4.1.Maven的仓库

                                                                                                                                                                      在 Maven 的术语中，仓库是一个位置。 项目中依赖的第三方库，这个库所在的位置叫做仓库。

                                                                                                                                                                      ​ Maven仓库有三种类型：本地仓库，中央仓库，远程仓库

                                                                                                                                                                      • 本地仓库

                                                                                                                                                                        在第一次执行 maven 命令的时候才被创建。默认情况下，会被创建在用户目录下 .m2/respository/ 目录，该目录即为本地仓库。 Maven所需要的任何构件都将存储在本地仓库中。

                                                                                                                                                                        修改本地仓库的目录地址，maven目录下conf/settings.xml文件，找到被注释的localRepository标签位置，添加配置

                                                                                                                                                                        <localRepository>D:\apache\maven\repository</localRepository>

                                                                                                                                                                      • 中央仓库

                                                                                                                                                                        由 Maven 社区提供的仓库，也可以称为官方库，其中包含了大量常用的库。服务器在国外。

                                                                                                                                                                      • 远程仓库 开发人员自己定制的仓库，包含了所需要的代码库或者其他工程中用到的 jar 文件。 配置远程仓库的地址，maven目录下conf/settings.xml文件，找到mirrors标签，修改

                                                                                                                                                                        <mirrors>
    <mirror>
          <id>alimaven</id>
          <name>aliyun maven</name>
          <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
          <mirrorOf>central</mirrorOf>
      </mirror>
      <mirror>
          <id>uk</id>
        <mirrorOf>central</mirrorOf>
          <name>Human Readable Name for this Mirror.</name>
        <url>http://uk.maven.org/maven2/</url>
      </mirror>
      <mirror>
          <id>CN</id>
          <name>OSChina Central</name>
          <url>http://maven.oschina.net/content/groups/public/</url>
          <mirrorOf>central</mirrorOf>
      </mirror>
      <mirror>
          <id>nexus</id>
          <name>internal nexus repository</name>
          <url>http://repo.maven.apache.org/maven2</url>
          <mirrorOf>central</mirrorOf>
      </mirror>
</mirrors>

                                                                                                                                                                      8.依赖范围（六种）

                                                                                                                                                                      Maven因为执行一系列编译、测试和部署运行等阶段操作，在不同的阶段下使用的依赖不同，依赖范围就是用来控制依赖与三种阶段（编译、测试、运行）的关系。 <scope>标签就是依赖范围的配置，默认是compile，可选配置还有test、provided、runtime、system、import

                                                                                                                                                                      <dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.8.2</version>
    <scope>test</scope>
</dependency>

                                                                                                                                                                      8.1.compile

                                                                                                                                                                      编译依赖范围。如果没有指定，就会默认使用该依赖范围。 使用此依赖范围的maven依赖，对于编译、测试、运行都有效。 如：fastjson，在编译、测试和运行的时候都需要使用该依赖。

                                                                                                                                                                      8.2.test

                                                                                                                                                                      测试依赖范围。只对于测试阶段有效，在编译主代码或者运行项目的使用时将无法使用此类依赖。 如：junit，它只在编译测试代码及运行测试的时候才需要。

                                                                                                                                                                      8.3.provided

                                                                                                                                                                      已提供依赖范围。对于编译和测试阶段有效，但在运行时无效。 如：servlet-api，编译和测试项目的时候需要该依赖，但在运行项目的时候，由于容器已经提供，就不需要重复地引入一遍。

                                                                                                                                                                      8.4.runtime

                                                                                                                                                                      运行时依赖范围。对于测试和运行阶段有效，但在编译主代码时无效。 如：jdbc驱动，项目主代码的编译只需要jdk提供的jdbc接口，只有在执行测试或者运行项目的时候才需要实现上述接口的具体jdbc驱动。

                                                                                                                                                                      8.5.import

                                                                                                                                                                      导入依赖范围。该依赖范围不会对三种阶段产生实际的影响。该依赖范围只能与dependencyManagement元素配合使用，其功能为将目标pom文件中dependencyManagement的配置导入合并到当前pom的dependencyManagement中。

                                                                                                                                                                      8.6.system

                                                                                                                                                                      系统依赖范围。该依赖与三种阶段的关系，和provided依赖范围完全一致。但是，使用system范围的依赖时必须通过systemPath元素显式的指定依赖文件的路径。由于此类依赖不是通过maven仓库解析的，而且往往与本机系统绑定，可能造成构建的不可移植。因此应该谨慎使用。

                                                                                                                                                                      9.1.JDK编译版本的插件

                                                                                                                                                                      ​ maven项目的默认编译版本是JDK1.5

                                                                                                                                                                      • 解决方式1：添加jdk编译插件

                                                                                                                                                                        <plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <version>3.2</version>
    <configuration>
        <source>1.8</source>
        <target>1.8</target>
        <encoding>utf-8</encoding>
    </configuration>
</plugin>

                                                                                                                                                                      • 解决方式2：添加properties属性 元素位置：project > properties

                                                                                                                                                                        <properties>
    <!-- 声明maven编译与打包使用的jdk版本 -->
    <maven.compiler.source>1.8</maven.compiler.source
      <maven.compiler.target>1.8</maven.compiler.target>
</properties>

                                                                                                                                                                      11.3.聚合与继承

                                                                                                                                                                      • 聚合 父模块 demo-parent 即为聚合模块，在 pom.xml 中使用 modules 元素聚合了所有的子元素。 聚合模块的打包方式必须为 pom，否则无法完成构建。 当在聚合模块上执行maven命令时，会根据各个子模块之间的依赖关系，依次在子模块上执行。

                                                                                                                                                                      • 继承 在项目的 pom.xml 文件中，通过 parent 元素来定义需要继承的父模块坐标信息。 可以直接继承到父模块中引入的依赖。 若父模块使用了 dependencyManagement 管理依赖，那么子元素不会自动继承被管理的依赖，可以选择需要继承的，在继承指定依赖时，会继承到父模块中已经配置的该依赖的版本号。

                                                                                                                                                                      ​ 在MyBatis核心配置文件中配置拦截器插件

                                                                                                                                                                      1.3.在代码中使用

                                                                                                                                                                      ​ 在查询列表前，使用 PageHelper.startPage(页码, 行数); 开启分页功能 ​ 使用 Page 接收查询列表，通过 getTotal() 获取数量，getResult() 获取列表

                                                                                                                                                                      @Override
public PageResult<Bill> listPage(BillPage, billPage){
    // 开启分页功能
    PageHelper.startPage(billPage.getPage(), billPage.getSize());
    // 查询用户列表
    Page<User> page = userMapper.listByParams(userParams);
    // 获取数量、列表
    return new PageResult<>((int) page.getTotal(), page.getResult);
}

                                                                                                                                                                      1.4.注意事项

                                                                                                                                                                      PageHelper.startPage方法重要提示

                                                                                                                                                                      只有紧跟在PageHelper.startPage方法后的第一个Mybatis的**查询（Select）**方法会被分页。

                                                                                                                                                                      请不要配置多个分页插件

                                                                                                                                                                      请不要在系统中配置多个分页插件(使用Spring时,mybatis-config.xml和Spring<bean>配置方式，请选择其中一种，不要同时配置多个分页插件)！

                                                                                                                                                                      分页插件不支持带有for update语句的分页

                                                                                                                                                                      对于带有for update的sql，会抛出运行时异常，对于这样的sql建议手动分页，毕竟这样的sql需要重视。

                                                                                                                                                                      分页插件不支持嵌套结果映射

                                                                                                                                                                      由于嵌套结果（一对多）方式会导致结果集被折叠，因此分页查询的结果在折叠后总数会减少，所以无法保证分页结果数量正确。
